# Пакет `risk`

## Назначение блока

**Риск-контур**: pre-trade и runtime проверки, торговые guard-ы, kill switch. Единая точка для отказа в исполнении и аварийной остановки.

## Почему он существует

Размазанные по коду проверки лимитов неаудируемы и обходятся случайно. Явный пакет **risk** гарантирует, что любое агрессивное действие прошло через согласованный pipeline.

## Основные обязанности

- pre-trade: лимиты размера, частоты, нотионала, времени;
- runtime: просадка, аномалии данных, нарушение инвариантов;
- kill switch: глобальный и селективный стоп торговли;
- генерация `domain.models.RiskEvent` для журнала.

## Что здесь должно находиться

- composable проверки и политики;
- состояние kill switch (in-memory + персистентный флаг при необходимости);
- интеграция с observability (алерты).

## Что здесь находиться не должно

- расчёт сигналов alpha;
- низкоуровневые HTTP вызовы (кроме чтения конфигурации из безопасного хранилища, если так решено архитектурно).

## Основные сущности / модули

| Модуль | Роль |
|--------|------|
| `pre_trade` | Проверки до отправки ордера. |
| `runtime` | Мониторинг во время работы. |
| `kill_switch` | Аварийный стоп. |
| `guards` | Фасад, собирающий правила. |

## Связи с другими блоками

- **execution** вызывает guards до и во время жизни позиций.
- **config** поставляет пороги.
- **control** может триггерить kill switch.
- **persistence** хранит события риска.

## Примеры будущего расширения

- динамические лимиты из внешнего risk-сервиса;
- per-strategy и per-instrument лимиты;
- корреляционные и портфельные проверки (через внешний контур).
