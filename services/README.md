# Пакет `services`

## Назначение блока

**Переиспользуемые прикладные сервисы**, не принадлежащие одному доменному агрегату: время, генерация идентификаторов, вспомогательные health-проверки.

## Почему он существует

Такие возможности нужны многим слоям; дублирование ломает тестируемость (replay требует подмены clock). Явные сервисы с интерфейсами упрощают подмену.

## Основные обязанности

- абстракция системного времени;
- генерация client order id / event id с гарантиями уникальности в рамках политики;
- мелкие утилиты без побочных эффектов на торговлю.

## Что здесь должно находиться

- `clock` — интерфейс времени;
- `id_generation` — фабрики id;
- `health` — вспомогательные проверки зависимостей (не HTTP-роуты).

## Что здесь находиться не должно

- бизнес-правила execution;
- прямой доступ к OKX;
- ORM.

## Связи с другими блоками

- **app**, **execution**, **replay** подменяют `Clock`.
- **persistence** может использовать генератор id для event store.

## Примеры будущего расширения

- распределённые UUID / snowflake id;
- монотонные часы для корреляции логов;
- кэш справочников инструментов.
