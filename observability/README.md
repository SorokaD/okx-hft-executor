# Пакет `observability`

## Назначение блока

**Наблюдаемость**: структурированное логирование, метрики, подготовка к трассировке, контракты алертов.

## Почему он существует

HFT/алго-контур без метрик и корреляции логов неэксплуатируем. Observability — обязательный слой, а не опция.

## Основные обязанности

- единый формат логов (JSON в проде);
- экспонирование технических и бизнес-метрик;
- hooks/spans для OpenTelemetry (без жёсткой привязки на каркасе);
- описание условий алертов (кодом или конфигом).

## Что здесь должно находиться

- инициализация логгера;
- обёртки над счётчиками/гистограммами;
- no-op реализации для локальной разработки.

## Что здесь находиться не должно

- основная бизнес-логика;
- секреты и PII в логах.

## Связи с другими блоками

- все пакеты используют общий logger/metrics registry;
- **control** отдаёт `/metrics` при интеграции с Prometheus.

## Примеры будущего расширения

- автокорреляция trace_id через run-loop;
- SLA по задержке данных и исполнения;
- интеграция с Grafana/Loki.
