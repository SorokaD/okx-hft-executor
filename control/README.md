# Пакет `control`

## Назначение блока

**Control plane**: HTTP (или иной) интерфейс для health, паузы торговли, resume, принудительного flatten, диагностики без пересборки ядра.

## Почему он существует

Операции «остановить торги сейчас» не должны требовать SSH и правки кода. Отдельный слой изолирует веб-стек от критического run-loop (опциональная зависимость `control` extra).

## Основные обязанности

- health/readiness endpoints;
- команды pause/resume/kill switch;
- безопасная аутентификация (вне каркаса — на стороне деплоя).

## Что здесь должно находиться

- тонкие адаптеры над `risk`, `execution`, `services.health`;
- сборка FastAPI/Starlette приложения (когда подключат extra).

## Что здесь находиться не должно

- тяжёлая стратегия;
- прямой OKX REST без прохождения через exchange пакет.

## Связи с другими блоками

- **risk.kill_switch** — цель оперативных команд;
- **observability** — метрики запросов control API;
- **app** — может поднять control server параллельно оркестратору.

## Примеры будущего расширения

- mTLS / API tokens;
- read-only диагностические дампы состояния (с редукцией секретов);
- интеграция с Kubernetes probes.
